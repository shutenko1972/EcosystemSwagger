4. Описание эндпоинтов:

Аутентификация (Auth)

1. `/api/auth/login` (POST)
Назначение: Вход пользователя в систему  
Проверяет:
- Корректность логина и пароля
- Наличие обязательных полей
Что делает:
- Создает новую сессию на 1 час
- Возвращает токен сессии
Вход: `login`, `password`  
Выход: `sessionToken`, `redirectUrl`

2. `/api/auth/logout` (POST)
Назначение: Выход пользователя из системы  
Проверяет:
- Существование сессии
Что делает:
- Удаляет сессию из активных
Вход: `sessionToken`  
Выход: `message` (статус выхода)

3. `/api/auth/check-session` (POST)
Назначение: Проверка валидности сессии  
Проверяет:
- Активность сессии
- Не истекло ли время сессии
Что делает:
- Возвращает информацию о сессии
Вход: `sessionToken`  
Выход: `valid`, `user_login`, `expires_at`

Чат (Chat)

4. `/api/chat/send` (POST)
Назначение: Отправка сообщения в AI-чат  
Проверяет:
- Валидность сессии
- Наличие текста сообщения
Что делает:
- Обрабатывает запрос через AI-модель
- Возвращает краткий ответ
Вход: `message`, `sessionToken`  
Выход: `answer` (ответ AI)

5. `/api/chat/clear` (POST)
Назначение: Очистка истории чата  
Проверяет:
- Валидность сессии
Что делает:
- Сбрасывает контекст чата
Вход: `sessionToken`  
Выход: `message` (статус очистки)

6. `/api/chat/copy` (POST)
Назначение: Копирование текста ответа  
Проверяет:
- Валидность сессии
- Наличие текста для копирования
Что делает:
- Подготавливает текст для копирования
Вход: `text`, `sessionToken`  
Выход: `message` (статус копирования)

Настройки модели (Settings)

7. `/api/settings/temperature` (POST)
Назначение: Установка параметра температуры AI-модели  
Проверяет:
- Валидность сессии
- Значение в диапазоне 0-200
Что делает:
- Устанавливает параметр случайности ответов
Вход: `value` (0-200), `sessionToken`  
Выход: `value` (подтверждение)

8. `/api/settings/topp` (POST)
Назначение: Установка параметра Top-P AI-модели  
Проверяет:
- Валидность сессии
- Значение в диапазоне 0-100
Что делает:
- Устанавливает параметр разнообразия ответов
Вход: `value` (0-100), `sessionToken`  
Выход: `value` (подтверждение)

Системные (System)

9. `/api/health` (GET)
Назначение: Проверка работоспособности сервера  
Проверяет:
- Доступность сервера
- Состояние служб
Что делает:
- Возвращает статус сервера
Вход: нет  
Выход: `status` (OK/Error)

10. `/api/info` (GET)
Назначение: Получение информации о сервере  
Проверяет: нет  
Что делает:
- Возвращает базовую информацию
Вход: нет  
Выход: `name`, `version`

Особенности безопасности:
- Все эндпоинты (кроме login, health, info) требуют `sessionToken`
- Сессии автоматически удаляются через 1 час
- Валидация входных параметров
- Логирование всех операций
